# Skill Routing Matrix

Use this matrix when adapting an old project's workflow skills into a new
project. Replace names with local equivalents where needed.

```text
Task kind
|-- Continue project / status-aware execution
|   +-- project entry skill
|   +-- state-lock pattern
|   +-- target preflight
|
|-- Route design / route switch / route explanation
|   +-- research-route-gate
|   +-- failure-attribution-strategy-bypass if old-route resemblance exists
|   +-- route-readout-card-authoring
|
|-- Experiment / rerun / manifest / benchmark
|   +-- experiment-launch-denominator-gate
|   +-- reusable-material-index
|   +-- research-route-gate
|   +-- checked output root and expected artifacts
|
|-- Failure result / weak diagnostic / suspicious signal
|   +-- failure-attribution-strategy-bypass
|   +-- living-engineering-error-loop if operational failure is possible
|   +-- route-readout-card-authoring
|
|-- Runtime / server / long job
|   +-- runtime-capability-inventory
|   +-- long-job-nightly-planning
|   +-- transfer/server guardrails
|   +-- engineering recurrence loop on repeated failures
|
|-- Subagent support work
|   +-- subagent coordination and active registry
|   +-- scoped support task only
|   +-- finalizer after completion
|
|-- Report / visual progress / project map
|   +-- status-reporting skill
|   +-- progress-map skill for explicit visual reports
|   +-- artifact/status sync only if files changed
|
|-- Closure / reusable artifact
|   +-- checked-result-finalizer
|   +-- status-artifact-registry-sync
|   +-- decision ledger
|
|-- Paper / claim writing
|   +-- paper-claim-writing
|   +-- checked-result-finalizer
|   +-- claim map
|
|-- Project OS repair / preservation
|   +-- project-os-bootstrap
|   +-- living-engineering-error-loop if a recurring process failure caused it
```

For each row, write:

```text
when to trigger:
minimal companion skills:
target authority files:
forbidden actions:
expected durable artifact:
```
