# Research Route Gate

Use this skill when the next action changes, evaluates, scales, or reports a
scientific or evidence-driven route. The main agent owns route judgment; support
agents may handle bounded audits, registry work, docs, transfer, or runner
support.

## Must Establish

Before an experiment or route change, identify:

```text
project entry rules:
current state:
route objective:
old-route resemblance:
mechanism difference:
fixed evidence object or denominator:
output root:
expected artifacts:
metric or evaluation signal:
pass:
hold:
stop:
scale-up condition:
forbidden next retry:
claim boundary:
finalization path:
```

Use the smallest useful fixed gate before scale-up. Query the project's
reusable material index, artifact registry, previous readouts, or equivalent
source before recreating material. If you skip lookup, state a concrete reason.

## Route Judgment

Use this split:

```text
pass:
  evidence meets the predeclared threshold and artifacts are checked.

hold:
  evidence is incomplete, suspicious, underpowered, or blocked by engineering.

stop:
  failure mechanism is clear enough that retrying the same layer is wasteful.

switch:
  the objective remains valid but should be reached by a different mechanism.
```

## Support Work

Support agents may handle:

```text
environment checks
artifact inventory
documentation drafts
registry sync
transfer/runtime preparation
```

Main agent retains:

```text
route judgment
metric interpretation
pass/hold/stop/switch
claim boundaries
state progression
```

## Forbidden

- Do not scale to full official evaluation before the smaller fixed gate passes.
- Do not claim metrics without fixed evidence object, artifact identity, metric
  definition, and finalization.
- Do not treat runtime, transfer, or runner errors as scientific evidence.
- Do not silently change branch, route, denominator, or mainline state.
- Do not retry a route that resembles a stopped route without stating the real
  mechanism difference.
